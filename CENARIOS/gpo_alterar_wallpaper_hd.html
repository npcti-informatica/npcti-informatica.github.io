GPO ALTERAR PAPEL DE PAREDE DA EQUIPE HELPDESK.


			CENÁRIO:

				A supervisão do SAC solicitou que fosse alterado o papel de parede exclusivamente para equipe do Helpdesk para diferenciar as equipes do SAC e do HD.

				Atualmente o papel de parede da equipe do HD é igual ao papel de parede da equipe do SAC;

					EQUIPE ATENDIMENTO SAC WALLPAPER

					

					EQUIPE ATENDIMENTO HELPDESK WALLPAPER

					
			

			 PROCEDIMENTOS COMPARTILHAMENTO DA IMAGEM:
				
				1.	Foram escolhidas os papeis de parede para equipe do Helpdesk onde as imagens foram salvas no diretório de exemplo 
				\\npcti-srvfs01\SUPORTE\PICTURE onde a pasta SUPORTE está configurada com as permissões:

				
					·	COMPARTILHAMENTO:

						

					·	SEGURANÇA:

						

					·	LOGO:

						

					·	PAPEL DE PAREDE:

						

				2.	Utilizando o programa BGINFO \\npcti-srvfs01\SUPORTE\BGINFO onde foi configurado um arquivo
				para mostrar dados do computador e o papel de parede;

					

				3.	No mesmo diretório do BGINFO foi criado um script.BAT para ser executado pela GPO;

					

			PROCEDIMENTOS COMPARTILHAMENTO DA IMAGEM: 				

				4.	No Domain Controller, após executar o GERENCIAMENTO DE POLITICA DE GRUPO será criada
				a política GPO_BGINFO_HD que será vinculada;

					

				5.	Após criar a política, “botão direito” > EDITAR para abrir o EDITOR DE GERENCIAMENTO DE POLITICA DE GRUPO;

					

				6.	Em seguida navegue até a opção CONFIGURAÇÃO DO USUÁRIO > POLITICAS > CONFIGURAÇÕES DO WINDOWS >
				SCRIPT LOGON/LOGOFF > LOGON;

					

				7.	Após abrir a política de LOGON na guia SCRIPTS clique em ADICIONAR;

					

				8.	No campo NOME DO SCRIPT foi informado o caminho de rede onde está o arquivo .BAT 
				que executa o script do BGINFO de configuração da imagem de fundo da equipe do helpdesk que 
				neste laboratório \\npcti-srvfs01\SUPORTE\BGINFO\npcti_hd.bat; 

					

				9.	Após adicionar a informação, clique em APLICAR e OK para salvar as alterações;

					

				10.	Feche o EDITOR DE GERENCIAMENTO DE POLITICA DE GRUPO e verifique na política na guia OPÇÕES
				se a regra foi configurada com sucesso; 

					

			
				
				OBSERVAÇÃO: Como a política para inserir um papel de parede exclusivo para o HELPDESK 
				não funcionou utilizando o recurso FILTRO DE SEGURANÇA da guia ESCOPO,
				foi necessário criar uma OU HELPDESK para que a GPO_bginfo_HD pudesse ser associada;

			

				
				
				->> A GPO vinculada a OU do Helpdesk <<--

				

				11.	No Domain Controller após executar o Powershell como administrador,
				foi executado o comando INVOKE-GPUPDATE para que a política de grupo seja aplicada imediatamente;

					

			PROCEDIMENTOS NO COMPUTADOR CLIENTE

				12. No computador cliente, foi executado o comando GPUPDATE /FORCE para atualizar a política de grupo que foi criada;

					

				13.	Em seguida foi realizado o LOGOFF/LOGON no computador cliente;

				14. Após o procedimento o papel de parede foi atualizado;
